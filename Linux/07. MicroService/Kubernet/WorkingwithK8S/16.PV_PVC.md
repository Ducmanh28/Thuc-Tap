# PersistentVolume và PersistentVolumeClaim (PVC)
## Định nghĩa (Definition):

PersistentVolume (PV): Là một phân vùng lưu trữ cấp cluster (Cluster-level resource) do Quản trị viên cấp phát thủ công (Static Provisioning) hoặc tự động tạo ra qua StorageClass (Dynamic Provisioning). PV độc lập với vòng đời của bất kỳ Pod nào sử dụng nó.

Ví dụ: 
```
---
apiVersion: v1                      # Sử dụng API version v1 của Kubernetes cho PersistentVolume.
kind: PersistentVolume              # Loại tài nguyên là PersistentVolume (PV) — ổ đĩa lưu trữ dùng cho Pod.
metadata:
  name: nginx-pv                    # Tên định danh PV này, dùng trong PVC thông qua 'volumeName'.
spec:
  capacity:
    storage: 2Gi                    # Dung lượng lưu trữ tối đa của PV này là 2GiB.
  accessModes:
    - ReadWriteOnce                # Cho phép một Pod gắn và ghi vào PV tại một thời điểm.
    - ReadWriteMany                # Cho phép nhiều Pod gắn và ghi vào PV cùng lúc (nếu hỗ trợ).
  persistentVolumeReclaimPolicy: Retain   # Sau khi PVC xóa, PV sẽ giữ lại dữ liệu và không bị xóa/tái sử dụng tự động.
  nfs:                              # Định nghĩa rằng PV này được mount từ hệ thống file NFS.
    server: 172.16.66.222           # Địa chỉ IP của máy chủ NFS đang chia sẻ thư mục.
    path: "/mnt/nfs"                # Đường dẫn thư mục chia sẻ trên máy chủ NFS.
```

PersistentVolumeClaim (PVC): Là một yêu cầu cấp phát lưu trữ từ phía người dùng. Pods yêu cầu tài nguyên lưu trữ (dung lượng, quyền truy cập) thông qua PVC. K8s Control Plane sẽ tự động tìm kiếm và Binding (kết nối) PVC với một PV thỏa mãn điều kiện.

Ví dụ:

```
apiVersion: v1                        # Phiên bản API của Kubernetes cho PVC.
kind: PersistentVolumeClaim          # Loại đối tượng là PersistentVolumeClaim — yêu cầu sử dụng PV.
metadata:
  name: nginx-pvc                    # Tên của PVC — dùng để tham chiếu trong Pod/Deployment.
  namespace: nginx-namespace         # PVC sẽ được tạo trong namespace 'nginx-namespace'.
spec:
  accessModes:                       # Cách truy cập volume.
    - ReadWriteOnce                  # Chỉ cho phép một Pod ghi/đọc volume tại một thời điểm.
  resources:
    requests:
      storage: 1Gi                   # Yêu cầu dung lượng lưu trữ là 1GiB.
  volumeName: nginx-pv               # Chỉ định sử dụng cụ thể PV có tên là 'nginx-pv'
```
Mối quan hệ giữa PV và PVC

- **Admin** tạo PV để cấp phát dung lượng.
- **Developer** tạo PVC để yêu cầu dung lượng.
- Kubernetes tự động **match** PVC với PV phù hợp theo tiêu chí:
    - Dung lượng ≥ yêu cầu.
    - Access mode tương thích.
    - Chưa được sử dụng.
## Thông số:

- `accessModes`:

  - `ReadWriteOnce` (RWO): Mount dạng read-write bởi một Node duy nhất.

  - `ReadOnlyMany` (ROX): Mount dạng read-only bởi nhiều Node.

  - `ReadWriteMany` (RWX): Mount dạng read-write bởi nhiều Node (thường cần NFS, CephFS).

- `persistentVolumeReclaimPolicy`:

  - `Retain`: Giữ lại PV và dữ liệu (yêu cầu dọn dẹp thủ công).

  - `Delete`: Tự động xóa PV và dữ liệu trên backend storage.

  - `storageClassName`: Lớp lưu trữ dùng để phân loại hoặc kích hoạt Dynamic Provisioning.

## Volume Lifecycle

- PVC có thể **reuse** lại PV nếu policy là `Retain`.
- Có 3 loại **reclaimPolicy** của PV:
    - `Retain`: Giữ lại dữ liệu sau khi PVC bị xóa.
    - `Delete`: Xóa volume backend luôn.
    - `Recycle` (deprecated): Dọn sạch dữ liệu.

**StorageClass** giúp Kubernetes **tự động tạo PV** khi bạn tạo PVC, không cần admin tạo trước thủ công.

Ví dụ PVC dùng StorageClass:

```YAML
apiVersion: v1                     # Phiên bản API của Kubernetes dùng để định nghĩa tài nguyên PVC.
kind: PersistentVolumeClaim        # Loại tài nguyên là PersistentVolumeClaim — yêu cầu sử dụng một PersistentVolume.
metadata:
  name: dynamic-pvc                # Tên của PVC — sẽ được dùng để gắn vào Pod hoặc Deployment.
spec:                              # Cấu hình chi tiết của PVC.
  accessModes:                     # Cách truy cập volume — định nghĩa quyền truy cập.
    - ReadWriteOnce                # Cho phép chỉ một Pod ghi/đọc volume tại một thời điểm.
  resources:
    requests:                      # Yêu cầu tài nguyên.
      storage: 2Gi                 # Kích thước lưu trữ yêu cầu là 2GiB.
  storageClassName: standard       # Chỉ định StorageClass — Kubernetes sẽ dùng class này để **dynamic provisioning**
```

## 5. Mount PVC vào Pod
Ví dụ mount PVC và 1 Pod

```YAML
apiVersion: v1                  # API version của Kubernetes dùng để định nghĩa tài nguyên Pod.
kind: Pod                       # Loại tài nguyên là Pod — đơn vị chạy nhỏ nhất trong Kubernetes.
metadata:
  name: mypod                   # Tên của Pod — dùng để quản lý và tham chiếu trong cluster.
spec:                           # Định nghĩa cấu hình cụ thể cho Pod.
  containers:                   # Danh sách các container bên trong Pod.
  - name: app                   # Tên container bên trong Pod.
    image: nginx                # Image container sử dụng — ở đây là nginx (phiên bản latest mặc định).
    volumeMounts:              # Gắn volume từ bên ngoài vào container.
    - mountPath: "/usr/share/nginx/html"   # Đường dẫn bên trong container nơi volume sẽ được mount.
      name: my-storage         # Tên volume cần gắn vào (phải khớp với tên volume khai báo bên dưới).
  volumes:                      # Danh sách các volumes mà Pod sử dụng.
  - name: my-storage            # Tên định danh nội bộ của volume — trỏ tới PVC bên dưới.
    persistentVolumeClaim:      # Loại volume là PersistentVolumeClaim (PVC) — yêu cầu dùng ổ đĩa đã định nghĩa.
      claimName: my-pvc         # Tên PVC đã được tạo từ trước — yêu cầu Kubernetes gắn volume này vào Pod.
```
