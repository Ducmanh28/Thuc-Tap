# ConfigMaps
ConfigMaps chỉ đơn giản là một kho lưu trữ khóa-giá trị giữ dữ liệu của bạn ở dạng văn bản thuần túy. 

Điều này trái ngược với Secret-cũng làm điều tương tự, nhưng mã hóa dữ liệu của bạn. 

Như tên của nó, bạn thường sẽ sử dụng ConfigMaps để lưu trữ các cấu hình tách biệt với ứng dụng thực tế của bạn. Bằng cách này, nếu bạn muốn thay đổi cấu hình (chẳng hạn như điểm cuối API mà pod của bạn đang gọi), bạn có thể thực hiện điều đó bằng cách thay đổi ConfigMap mà không cần phải thay đổi ứng dụng thực tế hoặc yaml triển khai, do đó loại bỏ nhu cầu triển khai lại ứng dụng của bạn một lần nữa

# ConfigMaps 101
ConfigMaps cần có cấu trúc chỉ định loại tài nguyên đó. Ví dụ: ConfigMap cho một đối tượng NGINX sẽ trông như thế này:
```YAML
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_PORT: "8080"
  APP_MODE: "production"
```
Khai báo Pod
```YAML
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: app
    image: nginx
    envFrom:
    - configMapRef:
        name: app-config
```
Như cách trên là chúng ta dùng ConfigMap như biến môi trường
```
root@k8s-master:~# kubectl apply -f configmap.yaml
```

Trong mỗi container thường có file riêng để cấu hình các biến, ta có thể mount volume configmap thay cho file đố
Tạo configmap
```
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  .env: |
    DB_HOST=mysql
    DB_USER=admin
    DB_PASS=123456
```
Tạo deployment
```
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: app
        image: my-app:latest
        volumeMounts:
        - name: config-volume
          mountPath: /app/.env    # mount trực tiếp thành file .env
          subPath: .env           # chỉ lấy key ".env"
      volumes:
      - name: config-volume
        configMap:
          name: app-config
```
Chúng ta chỉ nên dùng configmap để chứa 1 số biến cấu hình không quan trọng. Nếu bạn muốn lưu key, mật khẩu, token, 1 số biến nhạy cảm bạn nên dùng Secret
