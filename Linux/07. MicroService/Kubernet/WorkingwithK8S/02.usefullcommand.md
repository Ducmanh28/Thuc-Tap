# Usefull command trong K8S
## 1. Xác Thực Hạ Tầng 
Nhóm lệnh này dùng để đánh giá lớp vật lý/máy ảo và đảm bảo các node đã join vào cluster thành công, đồng thời kubelet đang giao tiếp tốt với kube-apiserver.
```
kubectl cluster-info
```
Mô tả: Trích xuất các endpoint quan trọng của Control Plane. Lệnh này xác nhận địa chỉ IP và port mà kube-apiserver cũng như KubeDNS/CoreDNS đang lắng nghe.

Ví dụ:
```
root@k8s-master:~# kubectl cluster-info
Kubernetes control plane is running at https://172.16.66.84:6443
CoreDNS is running at https://172.16.66.84:6443/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy

To further debug and diagnose cluster problems, use 'kubectl cluster-info dump'.
```

```
kubectl get nodes -o wide
```
Mô tả: Phân tích tổng quan topology của cluster.

Tham số kỹ thuật: Trợ từ -o wide cung cấp các metadata cực kỳ quan trọng đối với kỹ sư: Trạng thái node (Ready/NotReady), Internal IP (dùng cho mạng overlay), External IP, phiên bản kubelet/kube-proxy và đặc biệt là Container Runtime đang sử dụng (vd: containerd://1.6.x hoặc cri-o).

Ví dụ:
```
root@k8s-master:~# kubectl get nodes -o wide
NAME           STATUS   ROLES           AGE     VERSION    INTERNAL-IP    EXTERNAL-IP   OS-IMAGE             KERNEL-VERSION     CONTAINER-RUNTIME
k8s-master     Ready    control-plane   6d1h    v1.29.15   172.16.66.84   <none>        Ubuntu 24.04.4 LTS   6.8.0-64-generic   containerd://1.7.28
k8s-worker-1   Ready    <none>          5d23h   v1.29.15   172.16.66.83   <none>        Ubuntu 24.04.4 LTS   6.8.0-64-generic   containerd://1.7.28

```

## 2. Kiểm Chứng Control Plane Components
Trong kiến trúc K8s tiêu chuẩn, các thành phần não bộ (core components) thường được triển khai dưới dạng Static Pods và cô lập trong namespace kube-system. Việc kiểm tra không gian này giúp xác nhận hệ thống quản trị đang hoạt động bình thường.
```
kubectl get pods -n kube-system -o wide
```
Mô tả: Liệt kê các thành phần hạ tầng cốt lõi.

Phân tích đầu ra: Khi chạy lệnh này, bạn cần đảm bảo trạng thái Running của các thành phần sau:

- `kube-apiserver`: Điểm entrypoint duy nhất của K8s API.

- `etcd`: Backing store (key-value) lưu trữ toàn bộ state của cluster.

- `kube-scheduler`: Component chịu trách nhiệm tính toán và gán Pod vào các Worker Node phù hợp.

- `kube-controller-manager`: Control loop điều chỉnh trạng thái hiện tại (current state) về trạng thái mong muốn (desired state).

- Các DaemonSet phục vụ mạng (CNI plugin như Calico, Flannel) và kube-proxy.

Ví dụ:

## 3. Khám Phá Workload & Troubleshooting Cơ Bản
Khi bắt đầu triển khai ứng dụng, đây là bộ công cụ để phân tích trạng thái resource và gỡ lỗi (debug) ở mức độ cơ bản.
```
kubectl get all -A
```
Mô tả: Quét toàn bộ các resource object cơ bản (Pod, Service, DaemonSet, Deployment, ReplicaSet) trên tất cả các namespaces (-A). Dùng để có bức tranh tổng thể về hệ thống tại một thời điểm.

```
kubectl describe <resource_type> <resource_name> -n <namespace>
```
Mô tả: Trích xuất toàn bộ cấu hình (Spec), trạng thái (Status) và siêu dữ liệu (Metadata) của một object.

Ứng dụng chuyên sâu: Đây là công cụ số 1 để tìm nguyên nhân lỗi. Phần Events ở cuối kết quả trả về sẽ theo dõi chính xác các sự kiện ở mức hệ thống (ví dụ: FailedScheduling do thiếu CPU/RAM, hay FailedToPullImage do sai tag)

Ví dụ:
```
root@k8s-master:~# kubectl describe pod/etcd-k8s-master -n kube-system
Name:                 etcd-k8s-master
Namespace:            kube-system
Priority:             2000001000
Priority Class Name:  system-node-critical
Node:                 k8s-master/172.16.66.84
Start Time:           Mon, 23 Mar 2026 15:41:45 +0700
Labels:               component=etcd
                      tier=control-plane
```

Kiểm tra log:
```
kubectl logs <pod_name> -n <namespace> [-c <container_name>]
```
Mô tả: Truy xuất stdout/stderr từ container engine của Pod tương ứng.

Tham số kỹ thuật: Sử dụng -f (follow) để stream log realtime. Bắt buộc dùng -c nếu Pod áp dụng pattern multi-container (như Sidecar hoặc Init container). Thêm --previous để lấy log của instance container vừa bị crash trước đó.
```
kubectl exec -it <pod_name> -n <namespace> -- /bin/sh
```
Mô tả: Mở một interactive shell session trực tiếp vào namespace network và mount namespace của container.

Ứng dụng chuyên sâu: Cho phép kỹ sư đi sâu vào container để kiểm chứng phân quyền file system, đọc config map đã mount, hoặc test network routing (thông qua curl, nslookup) từ góc độ của chính Pod đó.

## 4. Triển Khai Declarative 
Mọi thao tác quản lý K8s thực tế đều dựa trên cơ chế Declarative (Khai báo) qua các file manifest YAML/JSON.
```
kubectl apply -f <file_manifest.yaml>
```
Mô tả: Gửi bản khai báo trạng thái mong muốn (Desired State) tới kube-apiserver để khởi tạo hoặc cập nhật các Kubernetes objects.
```
kubectl delete -f <file_manifest.yaml>
```
Mô tả: Gỡ bỏ các resource đã được định nghĩa trong file manifest ra khỏi cluster.
