# Làm việc với workload(Namespace,pod,container)
## Cấu trúc tổ chức tài nguyên trong Kubernetes

Kubernetes tổ chức tài nguyên theo mô hình phân tầng:

- **Namespace**: không gian tên, giúp phân tách tài nguyên 
- **Pod**: đơn vị nhỏ nhất chứa một hoặc nhiều container
- **Container**: nơi thực thi ứng dụng (image), chia sẻ mạng và volume trong Pod

Vậy chúng ta có thể suy ra mối quan hệ:

Namespace → Pod → Container

## Làm việc với Namespace

Tạo một namespace mới

Namespace là không gian logic để nhóm tài nguyên (pod, service...) lại

Để tạo namespace mới

```bash
kubectl create namespace <ten-namespace>
```

**Ví dụ:**

```bash
root@k8s-master:~# kubectl create namespace test
namespace/test created
```

**Xem danh sách các namespace**

```bash
root@k8s-master:~# kubectl get ns
NAME              STATUS   AGE
default           Active   6d3h
kube-flannel      Active   5d2h
kube-node-lease   Active   6d3h
kube-public       Active   6d3h
kube-system       Active   6d3h
test              Active   25s
```

Hoặc:

```bash
root@k8s-master:~# kubectl get namespaces
NAME              STATUS   AGE
default           Active   6d3h
kube-flannel      Active   5d2h
kube-node-lease   Active   6d3h
kube-public       Active   6d3h
kube-system       Active   6d3h
test              Active   44s
```

**Xem thông tin chi tiết của một namespace**

```bash
root@k8s-master:~# kubectl describe ns test
Name:         test
Labels:       kubernetes.io/metadata.name=test
Annotations:  <none>
Status:       Active

No resource quota.

No LimitRange resource.
```

**Xoá một namespace (cẩn thận!)**

```bash
kubectl delete ns <ten-namespace>
```

Lưu ý: Xóa namespace sẽ xóa tất cả tài nguyên bên trong nó.

## Làm việc với Pod

**Tạo một Pod đơn (dùng lệnh)**

```bash
kubectl run <ten-pod> --image=<image> --restart=Never -n <namespace>
```

**Ví dụ:**

```bash
root@k8s-master:~# kubectl run nginx-test --image=nginx --restart=Never -n test
pod/nginx-test created
```

Lưu ý: --restart=Never để tạo một pod độc lập, không nằm trong deployment.


**Xem danh sách các pod**

```bash
kubectl get pods -n <ten-namespace>

```

**Ví dụ:**

```bash
root@k8s-master:~# kubectl get pods -n test
NAME         READY   STATUS    RESTARTS   AGE
nginx-test   1/1     Running   0          35s
```

**Xem thông tin chi tiết của pod**

```bash
root@k8s-master:~# kubectl describe pod nginx-test -n test

Priority:         0

Labels:           run=nginx-test
Annotations:      <none>
Status:           Running

```

**Xoá một pod**

```bash
root@k8s-master:~# kubectl delete pod nginx-test -n test
pod "nginx-test" deleted
```


## Làm việc với Container 

Container luôn nằm bên trong pod, nên bạn thao tác với container qua pod

**Truy cập shell bên trong container**

```bash
kubectl exec -it <ten-pod> -n <ten-namespace> -- <shell>
```

**Ví dụ:**

```bash
root@k8s-master:~# kubectl exec -it nginx-test -n test -- sh
#
```

> Nếu image hỗ trợ bash thì dùng bash, còn không thì dùng sh.

## Làm việc với Deployment

Deployment giúp quản lý nhiều pod giống nhau, hỗ trợ tự động scale, cập nhật...

**Tạo deployment**

```bash
kubectl create deployment <ten-deployment> --image=<image> -n <namespace>
```

**Ví dụ:**

```bash
kubectl create deployment nginx-deploy --image=nginx -n test
```

**Xem danh sách deployment**

```bash
kubectl get deploy -n <ten-namespace>
```

**Mô tả chi tiết một deployment**

```bash
kubectl describe deploy <ten-deployment> -n <ten-namespace>
```

**Xem pod được tạo bởi deployment**

```bash
kubectl get pods -l app=<ten-deployment> -n <ten-namespace>
```

> -l dùng để lọc theo nhãn (label), mặc định deployment gán nhãn app=<tên>.
> 

**Xoá deployment**

```bash
kubectl delete deployment <ten-deployment> -n <ten-namespace>
```

## Một số lệnh khác

**Xem tất cả tài nguyên trong namespace**

```bash
kubectl get all -n <ten-namespace>
```

**Xem cấu trúc của một tài nguyên**

```bash
kubectl explain <loại-tài-nguyên>
```
