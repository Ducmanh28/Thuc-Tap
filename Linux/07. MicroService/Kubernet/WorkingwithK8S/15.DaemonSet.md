# 11. DeamonSet

## 1. DaemonSet trong Kubernetes là gì?

**DaemonSet** là một loại **controller** trong Kubernetes dùng để đảm bảo **mỗi Node trong cluster sẽ chạy ít nhất một Pod của DaemonSet**. Khi thêm Node mới, DaemonSet sẽ tự động tạo Pod trên Node đó. Ngược lại, khi Node bị xóa, Pod tương ứng cũng sẽ bị xóa.

DaemonSet thường được dùng cho các **agent hoặc daemon** cần chạy trên tất cả Node trong cluster.

Điểm khác biệt chính so với **Deployment** và **StatefulSet**:

**So sánh DaemonSet với các controller khác**

| **Controller** | **Khi nào dùng** |
| --- | --- |
| **Deployment** | Ứng dụng **stateless**, có thể scale lên/xuống dễ dàng |
| **StatefulSet** | Ứng dụng **stateful**, cần Persistent Volume và tên cố định |
| **DaemonSet** | Các **daemon hoặc agent** cần chạy trên mọi Node hoặc một tập Node cụ thể |

---

## 2. Các đặc điểm chính của DaemonSet

- **Một Pod trên mỗi Node**
    
    Mỗi Node sẽ chạy một Pod, đảm bảo tất cả Node đều được “giám sát” hoặc có agent hoạt động.
    
- **Triển khai tự động trên Node mới**
    
    Khi Node mới join cluster, DaemonSet sẽ tự động tạo Pod trên Node đó.
    
- **Có thể giới hạn Node**
    
    Sử dụng `nodeSelector`, `affinity` hoặc `tolerations` để chỉ định Pod chạy trên Node cụ thể.
    
- **Xóa Pod khi Node bị xóa**
    
    Pod của DaemonSet sẽ tự động bị xóa nếu Node bị remove hoặc nếu DaemonSet bị xóa.
    

---

## 3. Các trường hợp sử dụng DaemonSet

DaemonSet thường dùng cho các **ứng dụng daemon hoặc agent** chạy trên mọi Node:

- **Thu thập log**: Fluentd, Logstash, Datadog Agent.
- **Giám sát Node**: Prometheus Node Exporter, cAdvisor.
- **Quản lý mạng**: Calico, Cilium, Weave Net.
- **Quản lý bảo mật hoặc cấu hình Node**: Falco, Sysdig, các tool quản lý cấu hình.

## 4. DaemonSet101

```bash
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd-log-collector
  namespace: kube-system
  labels:
    app: fluentd
spec:
  selector:
    matchLabels:
      app: fluentd
  template:
    metadata:
      labels:
        app: fluentd
    spec:
      tolerations:
      # Cấu hình bỏ qua Taint của Control Plane để force schedule
      - key: node-role.kubernetes.io/control-plane
        operator: Exists
        effect: NoSchedule
      containers:
      - name: fluentd
        image: fluent/fluentd:v1.14-1
        resources:
          limits:
            memory: 200Mi
          requests:
            cpu: 100m
            memory: 200Mi
```

- Xem DaemonSet:

```bash
kubectl get daemonset -n kube-system
```

- Xem chi tiết DaemonSet:

```bash
kubectl describe daemonset kube-proxy -n kube-system
```
