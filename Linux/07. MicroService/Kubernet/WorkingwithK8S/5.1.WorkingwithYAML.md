# Thao tác với file YAML 
## Cấu trúc 4 tầng của một file Manifest 
Mọi object trong K8S (Pod, Service, Deployment,...) đều bắt buộc phải có 4 trường sau đây:

`apiVersion`: Xác định phiên bản của K8S API mà bạn đang sử dụng để tạo object này.

- **Ví dụ**: v1 (cho Pod, Service, ConfigMap), apps/v1 (cho Deployment, DaemonSet).

`kind`: Xác định loại tài nguyên bạn muốn tạo.

`metadata`: Chứa các dữ liệu giúp định danh object.
- `name`: Tên duy nhất của object trong một Namespace.
- `namespace`: Phân vùng logic (mặc định là default).
- `labels`: Các cặp Key-Value dùng để phân loại và giúp các thành phần khác tìm thấy nhau.

`spec`: Phần quan trọng nhất. Đây là nơi bạn khai báo Trạng thái mong muốn. Cấu trúc bên trong spec sẽ khác nhau tùy vào kind.

## Khai báo Pod 
Pod là đơn vị thực thi cơ bản. Trong spec của Pod, quan trọng nhất là mảng containers.

Thực hành: Khai báo Pod Nginx

Tạo file 1-pod.yaml:
```YAML
apiVersion: v1 # Pod thuộc core API
kind: Pod
metadata:
  name: web-pod
  namespace: test
  labels:
    app: my-web # Label này cực kỳ quan trọng cho bước sau
spec:
  containers: # Khai báo danh sách các container chạy trong Pod
  - name: nginx-main # Tên container
    image: nginx:1.21 # Image trên Registry
    ports:
    - containerPort: 80 # Cổng mà container lắng nghe bên trong Pod
```
Thực thi: `kubectl apply -f 1-pod.yaml`
Kiểm tra: `kubectl get pod web-pod -n test`

## Khai báo Deployment
Deployment không trực tiếp chạy Pod, nó quản lý một ReplicaSet, và ReplicaSet sẽ chạy Pod.

Lưu ý: Cơ chế Selector

Deployment dùng selector để biết nó cần quản lý những Pod nào. Giá trị của `selector.matchLabels` bắt buộc phải khớp với `template.metadata.labels`

Thực hành: Deployment với 3 bản sao

Tạo file 2-deploy.yaml:

```YAML
apiVersion: apps/v1 # Deployment thuộc nhóm apps API
kind: Deployment
metadata:
  name: web-deploy
  namespace: test
spec:
  replicas: 3 # Số lượng Pod muốn duy trì
  selector:
    matchLabels:
      app: my-web # Deployment tìm các Pod có nhãn này để quản lý
  template: # Đây là "khuôn mẫu" để sinh ra Pod mới
    metadata:
      labels:
        app: my-web # Phải khớp với selector ở trên
    spec:
      containers:
      - name: nginx-container
        image: nginx:1.21
        resources: # Khai báo giới hạn tài nguyên (Nâng cao)
          requests: # Mức cam kết tối thiểu để Scheduler xếp node
            cpu: "100m"
            memory: "128Mi"
          limits: # Mức trần tối đa, vượt quá sẽ bị OOMKill
            cpu: "200m"
            memory: "256Mi"
```
Thực thi: `kubectl apply -f 2-deploy.yaml`

Kiểm tra: `kubectl get deploy -n test` và xem 3 Pod được sinh ra

## Khai báo Service
Service cung cấp một IP cố định và cơ chế Load Balancing để truy cập vào nhóm Pod của Deployment.

Giải mã các loại Port trong Service:
- port: Cổng của Service (nằm bên trong Cluster)
- targetPort: Cổng của Pod mà Service sẽ forward traffic tới (thường là 80)
- nodePort: Cổng mở trên tất cả các Worker Node để truy cập từ ngoài (dải 30000-32767)

Thực hành: Expose ứng dụng qua NodePort

Tạo file 3-svc.yaml:
```YAML
apiVersion: v1
kind: Service
metadata:
  name: web-service
  namespace: test
spec:
  type: NodePort # Cho phép truy cập từ bên ngoài qua IP của Node
  selector:
    app: my-web # Service tìm các Pod có nhãn này để điều hướng traffic
  ports:
  - port: 8080 # App gọi Service qua cổng này
    targetPort: 80 # Service đẩy vào Pod qua cổng này
    nodePort: 32000 # Bạn truy cập trình duyệt qua cổng này
```
Thực thi: `kubectl apply -f 3-svc.yaml`

Truy cập: Mở trình duyệt gõ `http://172.16.66.82:32000` (IP của Worker 2)

### Health Checks & Affinity
Để ứng dụng chạy "sạch" và chuyên nghiệp, bạn cần khai báo cách K8S kiểm tra sức khỏe ứng dụng và nơi đặt Pod.

Thực hành: Khai báo Liveness Probe và NodeAffinity

Tạo file `4-advanced.yaml`:
```YAML
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-adv
  namespace: test
spec:
  replicas: 1
  selector:
    matchLabels:
      app: adv-web
  template:
    metadata:
      labels:
        app: adv-web
    spec:
      affinity: # Ép Pod chạy trên Worker Node
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
            - matchExpressions:
              - key: node-role.kubernetes.io/worker
                operator: In
                values: ["worker"]
      containers:
      - name: nginx
        image: nginx
        livenessProbe: # Kiểm tra xem app có bị "treo" không
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 5 # Đợi 5s sau khi khởi động mới check
          periodSeconds: 10 # Cứ 10s check 1 lần
        readinessProbe: # Kiểm tra xem app đã sẵn sàng nhận traffic chưa
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 5
```
