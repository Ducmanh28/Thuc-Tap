## Thiết lập cơ bản
Thực hiện trên cả 3 thiết bị
### Đổi hostname
Trong K8S, các node không chỉ liên lạc với nhau bằng IP mà còn bằng tên nên việc đổi tên sẽ giúp chúng ta dễ dàng thực hành sau này.

Thực hiện đổi tên các thiết bị
- U84: `hostnamectl set-hostname k8s-master`
- U83: `hostnamectl set-hostname k8s-worker-1`
- U82: `hostnamectl set-hostname k8s-worker-2`

### Cấu hình DNS
Không chỉ đổi tên, chúng ta sẽ sửa cả file `hosts` để các máy chủ có thể gọi tên nhau trực tiếp mà không cần một server DNS phức tạp

```
sudo tee -a /etc/hosts <<EOF
172.16.66.84 k8s-master
172.16.66.82 k8s-worker-1
172.16.66.83 k8s-worker-2
EOF
```
### Tắt Swap
K8s quản lý tài nguyên (RAM/CPU) cực kỳ khắt khe. Nếu hệ điều hành sử dụng Swap (dùng ổ cứng làm RAM ảo khi hết RAM), K8s sẽ không thể đo lường chính xác lượng RAM thực tế còn lại, dẫn đến việc phân bổ ứng dụng sai lệch hoặc làm hệ thống chậm đi đáng kể. 

```
sudo swapoff -a
sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab
```
### Cấu hình mạng
Trong K8s, các ứng dụng (Pod) giao tiếp qua một mạng ảo. Lớp mạng ảo này cần đi xuyên qua lớp mạng vật lý của Linux. Việc bật br_netfilter và ip_forward đảm bảo các gói tin mạng của Pod không bị hệ điều hành chặn lại và có thể được tường lửa (iptables) xử lý đúng các chính sách bảo mật mạng

```
# Bật các module kernel cần thiết
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF
sudo modprobe overlay
sudo modprobe br_netfilter
```
```
# Cấu hình sysctl để iptables có thể thấy bridged traffic
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system
```
## Cài đặt môi trường
Thực hiện trên cả 3 node
### Cài đặt `containerd`
K8s không tự chạy các container mà cần một phần mềm chuyên dụng. Containerd là lựa chọn tiêu chuẩn, siêu nhẹ và tối ưu. Việc thiết lập SystemdCgroup = true đảm bảo Containerd và hệ điều hành Ubuntu dùng chung một trình quản lý tài nguyên (systemd), tránh xung đột gây sập node

```
sudo apt-get update && sudo apt-get install -y containerd
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml > /dev/null
# Cấu hình dùng chung systemd cgroup
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/g' /etc/containerd/config.toml
sudo systemctl restart containerd
sudo systemctl enable containerd
```
### Cài đặt Kubeadm, Kubelet và Kubectl
Đây là bộ 3 công cụ của K8s:
- Kubeadm: Lệnh dùng để khởi tạo hoặc gia nhập cụm
- Kubelet: Chạy ngầm trên mọi node, nhận lệnh từ Master để khởi động container
- Kubectl: Công cụ dòng lệnh để bạn giao tiếp và điều khiển K8s

```
sudo apt-get install -y apt-transport-https ca-certificates curl gpg
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.29/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.29/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
```
**Lưu ý**: Để tránh vỡ cụm, chúng ta sử dụng `mark hold` để ngăn Ubuntu tự động cập nhật
## Khởi tạo cụm
Thực hiện trên `k8s-master`
```
sudo kubeadm init --control-plane-endpoint="172.16.66.84" --pod-network-cidr=10.244.0.0/16
```
Lệnh này biến máy `.84` thành "đầu não" điều phối. 
- Tham số `--control-plane-endpoint` khai báo IP quản lý chính.
- Tham số `--pod-network-cidr=10.244.0.0/16` khai báo dải IP ảo sẽ cấp phát cho các Pod sau này

Sau khi chạy xong, màn hình sẽ in ra một lệnh để sau này các worker-node sử dụng để join.
```
kubeadm join 172.16.66.84:6443 --token tsp3on.kz25z3vqlhx21isk \
        --discovery-token-ca-cert-hash sha256:3e409ebe0c7e4fb8ff21bd560fc31d39418c064a35fed3da28e5b6ca7ac4d115
```
