### B. Nguyên nhân gốc (Root Cause):

Node k8s-worker-2 (172.16.66.82) đang gặp sự cố crash CNI. Giao diện mạng ảo (Virtual Network Interface) flannel.1 và cni0 từ phiên hoạt động cũ vẫn còn tồn tại (zombie interface) trên kernel của hệ điều hành. Khi Flannel daemonset khởi động lại, quá trình cấp phát VTEP (VXLAN Tunnel Endpoint) gặp xung đột namespace/MAC address, báo lỗi file exists, dẫn đến chu kỳ CrashLoopBackOff và khiến kubelet đánh dấu node thành NotReady.

### C. Giải pháp khắc phục (Workaround/Fix):

Do chính sách BẢO MẬT THỰC THI (READ-ONLY), hệ thống từ chối quyền can thiệp vào interface OS. Yêu cầu Quản trị viên SSH trực tiếp vào k8s-worker-2 và thực thi chuỗi lệnh sau để dọn dẹp các stale interfaces:

```Bash
# 1. Thực thi trên node k8s-worker-2 (172.16.66.82) bằng quyền root:
ip link delete cni0
ip link delete flannel.1
rm -rf /var/lib/cni/
rm -f /etc/cni/net.d/*

# 2. Restart kubelet service trên k8s-worker-2:
systemctl restart kubelet

# 3. Thực thi trên node k8s-master (172.16.66.84) để force recreate Flannel pod:
kubectl delete pod -n kube-system -l app=flannel --field-selector spec.nodeName=k8s-worker-2
```

## B4. Kiểm chứng & Phân tích Routing: Pod -> Pod (Khác Node: k8s-worker-1 -> k8s-worker-2)

```Bash
> Thực thi: kubectl exec -it pod-a-w1 -- ping -c 2 10.244.2.5
```
```Plaintext
PING 10.244.2.5 (10.244.2.5) 56(84) bytes of data.
64 bytes from 10.244.2.5: icmp_seq=1 ttl=63 time=0.852 ms
64 bytes from 10.244.2.5: icmp_seq=2 ttl=63 time=0.615 ms
```
Phân tích Kỹ thuật (Inter-node Routing Path - Flannel VXLAN):
- Nguồn: pod-a-w1 (10.244.1.5). 
- Đích: pod-c-w2 (10.244.2.5).

Khác CIDR subnet. Kernel routing table trên k8s-worker-1 xác định subnet 10.244.2.0/24 phải được định tuyến qua interface flannel.1.

WorkFlow: 
- B1. pod-a-w1 (eth0) -> veth pair -> cni0 bridge (k8s-worker-1).
- B2. cni0 đẩy gói tin theo default route của host OS tới VTEP interface flannel.1.
- B3. L2 Encapsulation: Flannel daemon đóng gói (encapsulate) original L2 frame vào trong một UDP packet (Destination Port 8472).
- B4. UDP packet đi ra physical interface của host (VD: ens160 - 172.16.66.83), qua mạng LAN vật lý tới ens160 của k8s-worker-2 (172.16.66.82).
- B5. Kernel k8s-worker-2 nhận UDP packet ở port 8472, đẩy vào interface flannel.1 để gỡ bỏ vỏ (decapsulate).
- B6. Inner L2 packet xuất hiện tại flannel.1, được đẩy tiếp vào cni0 bridge của worker-2, và cuối cùng đi qua veth pair vào eth0 của pod-c-w2.
