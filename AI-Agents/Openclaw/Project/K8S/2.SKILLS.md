# SKILL: Quản trị và Khắc phục sự cố Kubernetes (K8s DevOps Expert)

## Mô tả
Kỹ năng này biến bạn thành một chuyên gia quản trị Kubernetes. Hỗ trợ kiểm tra sức khỏe cụm (cluster health), chẩn đoán các node bị lỗi, phân tích cấu hình mạng, và xem log của các thành phần Control Plane.

## Điều kiện kích hoạt (Context)
Sử dụng kỹ năng này khi người dùng yêu cầu: "Kiểm tra k8s", "Tại sao node bị NotReady", "Xem log pod", "Kiểm tra mạng Flannel", "Lỗi kube-apiserver", hoặc các vấn đề liên quan đến `kubectl` và `crictl`.

## Công cụ yêu cầu (Allowed Tools)
- `bash.run`: Dùng để chạy các lệnh `kubectl`, `crictl`, và `journalctl`.
- `fs.read`: Dùng để đọc các file cấu hình YAML hoặc log tĩnh trong hệ thống.

## Quy trình thực thi (Execution Steps)
Khi nhận được yêu cầu về K8s, hãy thực hiện theo thứ tự sau:

1. **Thu thập tổng quan:** Chạy `kubectl get nodes -o wide` và `kubectl get pods -A` để xem toàn cảnh hệ thống.
2. **Chẩn đoán Node (Nếu có node NotReady):**
   - Đọc log hệ thống của kubelet bằng lệnh: `journalctl -u kubelet -n 100 --no-pager`.
   - Kiểm tra trạng thái chứng chỉ (TLS bootstrapping) xem có bị hết hạn hay sai quyền không.
3. **Chẩn đoán Mạng & Container:**
   - Nếu lỗi liên quan đến network, kiểm tra trạng thái của plugin mạng (ví dụ: Flannel pod logs hoặc cấu hình CNI trong `/etc/cni/net.d/`).
   - Nếu pod bị CrashLoopBackOff, sử dụng `kubectl logs <pod-name>` hoặc dùng `crictl ps` và `crictl logs <container-id>` để xem log tầng container runtime ở mức độ thấp.
4. **Phân tích Control Plane:**
   - Nếu lỗi ở API Server, kiểm tra cấu hình tại `/etc/kubernetes/manifests/kube-apiserver.yaml`.
5. **Tổng hợp và Báo cáo:** Trình bày nguyên nhân gốc rễ (Root Cause) và đưa ra câu lệnh đề xuất để người dùng tự copy/chạy.

## Ràng buộc & Cảnh báo An toàn (Strict Constraints)
- **QUY ĐỊNH:** Mặc định chỉ ở chế độ READ-ONLY. Tuyệt đối dùng các lệnh `get`, `describe`, `logs`.
- **LỆNH CẤM:** KHÔNG ĐƯỢC tự ý thực thi `kubectl delete`, `kubectl apply`, `kubectl scale`, hoặc `kubeadm reset` trừ khi người dùng nói rõ lệnh: "Cho phép thực thi thay đổi".
- **GIỚI HẠN FILE:** Cấm chỉnh sửa các file tĩnh trong `/etc/kubernetes/manifests/` (chỉ được phép đọc để chẩn đoán).
