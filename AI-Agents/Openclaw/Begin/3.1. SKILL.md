# `SKILL.md` bản chất là gì?
Mặc dù các mô hình ngôn ngữ (như Gemini 1.5 Flash) rất thông minh, nhưng mặc định chúng chỉ là những "bộ não" có kiến thức tổng quát. Chúng không biết quy trình làm việc cụ thể của riêng bạn.

Trong OpenClaw, SKILL.md chính là bản SOP (Standard Operating Procedure - Quy trình Thao tác Chuẩn) dành cho AI.
Nó là một file văn bản định dạng Markdown, dùng để hướng dẫn Agent (đặc vụ AI) cách thực hiện một nhiệm vụ cụ thể từng bước một, khi nào nên dùng công cụ nào (đọc file, gọi API, duyệt web), và định dạng kết quả trả về ra sao.

# Cấu trúc của một file SKILL.md
Một file SKILL.md mạnh mẽ và không bị AI hiểu lầm thường bao gồm 5 phần cốt lõi:

Tên và Mô tả (Name & Description): Giới thiệu ngắn gọn kỹ năng này làm gì để hệ thống phân loại.

Điều kiện kích hoạt (Triggers/Context): Khi nào AI nên sử dụng kỹ năng này? (Ví dụ: "Khi người dùng yêu cầu tóm tắt báo cáo").

Công cụ yêu cầu (Required Tools): Liệt kê các công cụ (tools) mà AI được phép dùng trong kỹ năng này (ví dụ: fs.read, browser.search, bash.run).

Quy trình từng bước (Execution Steps): Hướng dẫn chi tiết 1-2-3 để AI làm theo, tránh việc AI tự suy diễn lan man.

Quy định đầu ra (Output Format): Ràng buộc cách AI trình bày kết quả (JSON, Markdown, bảng biểu...).

# Mẫu Ví dụ Thực tế: Kỹ năng "Phân tích và Tóm tắt Log"
Dưới đây là một mẫu SKILL.md hoàn chỉnh. Bạn có thể tạo một file tên là Analyze_Logs.md trong thư mục workspace của OpenClaw.

```Markdown
# SKILL: Phân tích và Tóm tắt Log Hệ thống

## Mô tả
Kỹ năng này giúp đọc các file log `.txt` hoặc `.log`, tìm kiếm các thông báo lỗi (Error/Warning), và xuất ra một báo cáo ngắn gọn.

## Điều kiện kích hoạt
Sử dụng kỹ năng này khi người dùng yêu cầu: "Kiểm tra log", "Tìm lỗi trong file", hoặc "Tóm tắt tình trạng hệ thống".

## Công cụ cho phép (Allowed Tools)
- `fs.read`: Để đọc nội dung file log.
- `bash.run`: Để chạy lệnh `grep` lọc từ khóa nhanh (nếu file quá lớn).

## Quy trình thực thi (Steps)
1. Xác định đường dẫn file log mà người dùng cung cấp.
2. Dùng công cụ đọc file để lấy nội dung. Nếu file lớn hơn 1000 dòng, ưu tiên dùng lệnh `grep -i "error\|warn"` để lọc trước.
3. Phân tích nội dung vừa đọc, nhóm các lỗi giống nhau lại thành một cụm.
4. Xác định nguyên nhân gốc rễ (Root Cause) có thể xảy ra nhất dựa trên kiến thức của bạn.
5. Viết báo cáo trả về cho người dùng.

## Ràng buộc & Đầu ra (Constraints & Output)
- KHÔNG được tự ý sửa hay xóa file log.
- KHÔNG dịch các đoạn mã lỗi sang ngôn ngữ khác, giữ nguyên text gốc của log.
- Cấu trúc trả về bắt buộc phải dùng thẻ Markdown như sau:
  ### 🚨 Báo cáo Lỗi Hệ thống
  - **Tổng số lỗi:** [Số lượng]
  - **Các lỗi chính:**
    1. [Tên lỗi 1] - [Số lần lặp lại]
  - **Chẩn đoán nguyên nhân:** ...
  - **Đề xuất khắc phục:** ...
```
# Cách nạp SKILL vào hệ thống OpenClaw
Sau khi bạn đã viết xong file SKILL.md, bạn cần nói cho OpenClaw biết để nó "học" kỹ năng này.

Lưu file: Đặt file .md này vào thư mục làm việc mặc định của OpenClaw (thường là thư mục workspace hoặc thư mục memory mà bạn đã cấu hình).

Cập nhật bộ nhớ (Reindex): Khởi chạy terminal và sử dụng lệnh quản lý bộ nhớ của OpenClaw để AI đọc và lập chỉ mục file mới:

```Bash
openclaw memory reindex
```
Kiểm tra kỹ năng: Chạy lệnh rà soát để đảm bảo OpenClaw đã nhận diện được kỹ năng và không bị thiếu công cụ (Missing requirements):

```Bash
openclaw skills inspect
```
