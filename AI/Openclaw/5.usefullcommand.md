# Cẩm nang Dòng lệnh OpenClaw (CLI Reference)
Công cụ openclaw được thiết kế linh hoạt, cho phép bạn điều khiển toàn bộ vòng đời của trợ lý AI, từ cấu hình mạng, quản lý model, đến giao tiếp qua các kênh chat.

## 1. Tùy chọn Toàn cục (Global Options)
Các cờ (flags) này có thể đi kèm với bất kỳ lệnh nào để thay đổi hành vi mặc định của CLI.

### `--container <name>`: 
Ép CLI chạy bên trong một container Podman/Docker cụ thể.

**Ví dụ**: `openclaw status --container my-openclaw-bot`

### `--dev`: 
Khởi chạy ở chế độ Developer. Cách ly dữ liệu vào ~/.openclaw-dev và chuyển cổng Gateway sang 19001.

**Ví dụ**: `openclaw gateway --dev`

### `--log-level <level>`: 
Ép hệ thống ghi log theo mức độ ưu tiên (silent, fatal, error, warn, info, debug, trace).

**Ví dụ**: `openclaw gateway --log-level debug`

### `--no-color`: 
Tắt định dạng màu ANSI trong terminal (hữu ích khi xuất log ra file text).

**Ví dụ**: `openclaw doctor --no-color > report.txt`

### `--profile <name>`: 
Sử dụng một profile cấu hình riêng biệt, tạo ra các "nhân bản" cấu hình độc lập.

**Ví dụ**: `openclaw gateway --profile trading-bot`

### `-h, --help`: 
Hiển thị trợ giúp cho CLI hoặc cho một lệnh cụ thể.

### `-V, --version`: 
Hiển thị phiên bản hiện tại của OpenClaw.

## 2. Nhóm Lệnh Vận hành Cốt lõi (Core Operations)
Quản lý "trái tim" của hệ thống - Gateway và Node.

### `gateway`: 
Khởi chạy và quản lý máy chủ WebSocket Gateway.

Hướng dẫn: Dùng để chạy bot ngầm. Thêm `--force` để tự động giải phóng cổng nếu bị kẹt.

**Ví dụ**: `openclaw gateway --force`
### `tui`: 
Mở Giao diện người dùng trên Terminal (Text-based UI).

Hướng dẫn: Dùng để trò chuyện và theo dõi suy nghĩ của Agent trực tiếp trên màn hình đen.

**Ví dụ**: `openclaw tui`

### `agent`: 
Chạy một chu kỳ (turn) của agent thông qua Gateway.

Hướng dẫn: Dùng để gửi lệnh trực tiếp cho AI xử lý và thoát.

**Ví dụ**: `openclaw agent --message "Tóm tắt file log hôm nay"`

### `node`: 
Chạy dịch vụ máy chủ Node không giao diện (headless).

**Ví dụ**: `openclaw node start`

### `daemon`: 
Bí danh (alias) cũ của lệnh khởi chạy gateway.

**Ví dụ**: `openclaw daemon`

## 3. Nhóm Lệnh Cấu hình & Cài đặt (Config & Setup)
Thiết lập môi trường, API keys và thông số hệ thống.

### `config`: 
Đọc hoặc ghi cấu hình mà không cần giao diện tương tác.

Hướng dẫn: Rất tốt cho các tập lệnh tự động hóa (bash script).

**Ví dụ**: `openclaw config set agent.model "google/gemini-1.5-flash"`

### `configure`: 
Mở trình hướng dẫn cài đặt tương tác từng bước (Interactive).

**Ví dụ**: `openclaw configure`

### `setup`: 
Khởi tạo thư mục cấu hình và workspace ban đầu.

**Ví dụ**: `openclaw setup`

### `onboard`: 
Khởi chạy luồng hướng dẫn cho người mới (onboarding) về gateway và skills.

**Ví dụ**: `openclaw onboard`

### reset: 
Xóa toàn bộ cấu hình và dữ liệu cục bộ (Đưa về trạng thái xuất xưởng, nhưng vẫn giữ CLI).

**Ví dụ**: `openclaw reset`

### uninstall: 
Xóa triệt để các dịch vụ chạy ngầm và dữ liệu của Gateway.

**Ví dụ**: `openclaw uninstall`

## 4. Nhóm Lệnh Giám sát & Bắt bệnh (Monitoring & Diagnostics)
Theo dõi sức khỏe và xử lý sự cố.

### `status`: 
Hiển thị bảng tổng quan về sức khỏe, các kênh đang kết nối và phiên làm việc.

**Ví dụ**: `openclaw status --deep`

### `doctor`: 
Tự động rà soát hệ thống tìm lỗi cấu hình và đề xuất cách sửa.

Hướng dẫn: Luôn chạy lệnh này sau khi cập nhật hoặc đổi cấu hình mạng.

**Ví dụ**: `openclaw doctor --fix`

### `health`: 
Trích xuất dữ liệu health-check thuần túy từ Gateway.

**Ví dụ**: `openclaw health`

### `logs`: 
Đọc log thời gian thực của gateway thông qua chuẩn RPC.

**Ví dụ**: `openclaw logs --follow`

### `dashboard`: 
Mở giao diện điều khiển (Web UI) trên trình duyệt bằng token hiện tại.

**Ví dụ**: `openclaw dashboard`

### `system`: 
Xem các sự kiện hệ thống cốt lõi và nhịp tim (heartbeat) của tiến trình.

**Ví dụ**: `openclaw system events`

## 5. Nhóm Lệnh Giao tiếp & Kênh (Channels & Comms)
Quản lý sự tương tác với thế giới bên ngoài (Telegram, Discord, Webhooks).

### `channels`: 
Quản lý các kênh chat (đăng nhập, cấu hình, kiểm tra).

**Ví dụ**: `openclaw channels login --verbose`

### `message`: 
Gửi, đọc hoặc quản lý tin nhắn thông qua các kênh đã kết nối.

**Ví dụ**: `openclaw message send --channel telegram --target @mychat --message "Xin chào!"`

### `webhooks`: 
Cấu hình các endpoint để nhận dữ liệu từ các dịch vụ bên thứ ba (như TradingView).

**Ví dụ**: `openclaw webhooks list`

### `directory`: 
Tra cứu ID liên hệ và group trong các mạng lưới đã kết nối.

**Ví dụ**: `openclaw directory search "Crypto Group"`

## 6. Nhóm Quản lý Sinh thái AI (AI Ecosystem)
Kiểm soát hành vi, "bộ não" và kỹ năng của bot.

### `models`: 
Khám phá, quét và cấu hình các nhà cung cấp mô hình ngôn ngữ (LLM).

**Ví dụ**: `openclaw models list`

### `agents`: 
Quản lý các đặc vụ độc lập (cấu hình luồng dữ liệu, xác thực).

**Ví dụ**: `openclaw agents list`

### `skills`: 
Xem và quản lý các kỹ năng (mã code/chức năng) mà agent được phép dùng.

**Ví dụ**: `openclaw skills inspect`

### `plugins`: 
Quản lý các phần mở rộng (extensions) của hệ thống.

**Ví dụ**: `openclaw plugins enable memory-core`

### `memory`: 
Tìm kiếm và lập chỉ mục lại hệ thống lưu trữ bộ nhớ (Vector DB) của agent.

**Ví dụ**: `openclaw memory reindex`

### `hooks`: 
Quản lý các điểm móc nối (hooks) nội bộ của agent.

**Ví dụ**: `openclaw hooks list`

### `mcp`: 
Quản lý các máy chủ MCP (Model Context Protocol) nhúng.

**Ví dụ**: `openclaw mcp status`

### `sandbox`: 
Cấu hình container cách ly để bảo vệ host khi agent chạy code.

**Ví dụ**: `openclaw sandbox start`

## 7. Nhóm Bảo mật & Quản lý Truy cập (Security & Access)
### `security`: 
Đánh giá an ninh hệ thống và kiểm tra cấu hình.

**Ví dụ**: `openclaw security audit --deep`

### `secrets`: 
Quản lý và tải lại các biến bí mật (API keys, passwords) lúc runtime.

**Ví dụ**: `openclaw secrets reload`

### `approvals`: 
Quản lý cơ chế phê duyệt khi agent muốn thực thi lệnh nhạy cảm.

**Ví dụ**: `openclaw approvals list`

### `devices`: 
Quản lý ghép nối thiết bị và quản lý token ủy quyền.

**Ví dụ**: `openclaw devices list`

### `pairing`: 
Phê duyệt yêu cầu kết nối an toàn (Direct Message).

**Ví dụ**: `openclaw pairing approve`

## 8. Nhóm Công cụ Tiện ích (Utilities & Misc)
### `browser`: 
Điều khiển phiên bản Chrome/Chromium ẩn danh do OpenClaw quản lý.

**Ví dụ**: `openclaw browser restart`

### `cron`: 
Đặt lịch các tác vụ định kỳ qua bộ lập lịch Gateway.

**Ví dụ**: `openclaw cron list`

### `dns`: 
Cấu hình mạng lưới tìm kiếm diện rộng (hỗ trợ Tailscale/CoreDNS).

**Ví dụ**: `openclaw dns status`

### `backup`: 
Tạo và xác minh file nén sao lưu toàn bộ trạng thái hệ thống.

**Ví dụ**: `openclaw backup create`

### `sessions`: 
Liệt kê các phiên hội thoại đang mở hoặc đã lưu.

**Ví dụ**: `openclaw sessions list`

### `nodes`: 
Quản lý việc ghép nối và ra lệnh cho các node ngoại vi.

**Ví dụ**: `openclaw nodes list`

### `qr`: 
Tạo mã QR để cài đặt và ghép nối thiết bị iOS.

**Ví dụ**: `openclaw qr`

### `update`: 
Cập nhật mã nguồn OpenClaw lên phiên bản mới nhất.

**Ví dụ**: `openclaw update`

### `completion`: 
Sinh mã cho bash/zsh để tự động điền lệnh bằng phím Tab.

**Ví dụ**: `openclaw completion >> ~/.bashrc`

### `docs`: 
Tìm kiếm tài liệu trực tiếp trên terminal.

**Ví dụ**: `openclaw docs "how to setup telegram"`

### `acp`: 
Các công cụ cho Agent Control Protocol.

**Ví dụ**: `openclaw acp status`

### `clawbot`: 
Bí danh dùng để tương thích ngược với hệ thống cũ.

## 9. Một số câu lệnh hay được sử dụng nhất
Mở terminal:
```
openclaw tui
```

Theo dõi trạng thái:
```
openclaw gateway status
openclaw status
```

Tạm dừng dịch vụ:
```
openclaw gateway stop
```

Sửa chữa:
```
openclaw doctor
openclaw doctor --fix
```
Kiểm tra cấu hình
```
openclaw config
```
